Summary Documentation¶

Entry Point - Concept and Model¶

  • Concept - The RAM Concept process serving the API.

  • Model - The in-memory data in the the Concept process.

RAM Concept Data Model¶

The outline below shows the containment hierarchy of the data within a Model.

  • Concretes - The container for all Concrete mixes.

    • Concrete - A concrete mix.

  • StrandMaterials - The container for all StrandMaterial s.

    • StrandMaterial - A Strand Material.

  • DuctSystems - The container for all DuctSystem s.

    • DuctSystem - A Duct System.

  • AnchorSystems - The container for all AnchorSystem s.

    • AnchorSystem - An Anchor System.

  • PTSystems - The container for all PTSystem s.

    • PTSystem - A PT System.

  • Signs - Settings for API (and user) sign conventions.

  • Units - Settings for API (and user) units.

  • CalcOptions - Settings for meshing and calculations.

  • CadManager - The manager of RAM Concept’s CAD system.

    • StructureLayer - There is always exactly 1 Structure (“Mesh Input”) layer

      • Beam - Beams on the StructureLayer

      • Column - Columns on the StructureLayer (tracked by above/below slab)

      • SlabArea - SlabAreas on the StructureLayer

      • SlabOpening - SlabOpenings on the StructureLayer

      • Wall - Walls on the StructureLayer (tracked by above/below slab)

      • PointSpring - PointSprings on the StructureLayer

      • LineSpring - LineSprings on the StructureLayer

      • AreaSpring - AreaSprings on the StructureLayer

      • PointSupport - PointSupports on the StructureLayer

      • LineSupport - LineSupports on the StructureLayer

    • ElementLayer - There is always exactly 1 Element layer. It’s contents are read-only.

      • ColumnElement - ColumnElement on the ElementLayer (tracked by above/below slab)

      • SlabElement - SlabElements on the ElementLayer

      • WallElement - WallElements on the ElementLayer (tracked by above/below slab)

      • WallElementGroup - WallElementGroups on the ElementLayer (tracked by above/below slab)

      • PointSpring - PointSprings on the ElementLayer

      • LineSpring - LineSprings on the ElementLayer

      • AreaSpring - AreaSprings on the ElementLayer

      • PointSupport - PointSupports on the ElementLayer

      • LineSupport - LineSupports on the ElementLayer

    • ForceLoadingLayer - Multiple ForceLoadingLayers (always Self-Dead, Balance, Hyperstatic + any editable ForceLoadingLayers)

      • PointLoad - PointLoads on the ForceLoadingLayer

      • LineLoad - LineLoads on the ForceLoadingLayer

      • AreaLoad - AreaLoads on the ForceLoadingLayer

    • ShrinkageLoadingLayer - Any number of ShrinkageLoadingLayers

      • ShrinkageAreaLoad - ShrinkageAreaLoads on the ShrinkageLoadingLayer

    • TemperatureLoadingLayer - Any number of TemperatureLoadingLayers

      • TemperatureAreaLoad - TemperatureAreaLoads on the TemperatureLoadingLayer

    • LoadComboLayer - Any number of LoadComboLayers

      • LoadFactor - LoadFactors for the LoadComboLayer

    • TendonLayer - The 4 TendonLayers (manual, generated) x (longitude, latitude).

      • TendonSegment - TendonSegments on the TendonLayer

      • Jack - Jacks on the TendonLayer

      • TendonNode - TendonNodes on the TendonLayer (added and removed automatically)

    • DefaultBeam - The singleton settings for Beams to be added.

    • DefaultColumn - The singleton settings for Columns to be added.

    • DefaultSlabArea - The singleton settings for SlabAreas to be added.

    • DefaultSlabOpening - The singleton settings for SlabOpenings to be added.

    • DefaultWall - The singleton settings for Walls to be added.

    • DefaultPointSpring - The singleton settings for PointSprings to be added.

    • DefaultLineSpring - The singleton settings for LineSprings to be added.

    • DefaultAreaSpring - The singleton settings for AreaSprings to be added.

    • DefaultPointSupport - The singleton settings for PointSupports to be added.

    • DefaultLineSupport - The singleton settings for LineSupports to be added.

    • DefaultPointLoad - The singleton settings for PointLoads to be added.

    • DefaultLineLoad - The singleton settings for LineLoads to be added.

    • DefaultAreaLoad - The singleton settings for AreaLoads to be added.

    • DefaultShrinkageAreaLoad - The singleton settings for ShrinkageAreaLoads to be added.

    • DefaultTemperatureAreaLoad - The singleton settings for TemperatureAreaLoads to be added.

    • DefaultTendonSegment - The singleton settings for TendonSegments to be added.

    • DefaultJack - The singleton settings for Jacks to be added.

Utility Classes¶

These classes assist in creating, setting and accessing the data:

  • Point2D - An immutable 2D point.

  • Point3D - An immutable 3D point.

  • LineSegment2D - An immutable 2D line segment.

  • Polygon2D - An immutable 2D polygon.

Abstract Data Classes¶

There is generally little need to understand or directly work with these abstract classes. The hierarchy below shows the class inheritance.

  • Data - Superclass for all the objects contained in the Model.

    • CadLayer - A layer in RAM Concept’s CAD system.

      • EnvelopResultLayer - Superclass for FullResultLayer

        • FullResultLayer - Superclass for LoadingLayer, and LoadComboLayer

          • LoadingLayer - Superclass for ForceLoadingLayer, TemperatureLoadingLayer and ShrinkageLoadingLayer

    • CadEntity - An entity (residing in a layer) in RAM Concept’s CAD system.

      • ConcreteMember - A concrete member on the StructureLayer.

        • ConcreteSupport - A concrete support (Wall or Column).

        • ConcreteSpanningMember - A concrete spanning member (SlabArea or Beam).

      • ForceLoad - A load on a ForceLoadingLayer.

      • Spring - A spring on the StructureLayer or ElementLayer.

      • Element - An Element on the ElementLayer.
