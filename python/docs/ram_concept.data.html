ram_concept.data module¶

class Data(uid, model)¶

Bases: object

Data represents a significant data object in a Model.

Direct use of this class (instead of a more-specific subclass) is likely an error.

This class should only be subclassed by the framework.

Parameters:

  • uid (int) –

  • model (Model) –

property model¶

The Model that contains this Data.

property name: str¶

Name of this Data (note that name is not relevant in all Data subclasses)

Type:

property number: int¶

1-based Number of this Data (note that number is not relevant in all Data subclasses)

Read-only UID of this Data
