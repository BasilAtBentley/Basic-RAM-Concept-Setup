class LoadComboAnalysisType(value)¶

Bases: Enum

The analysis type of a LoadComboLayer.

The available values are:

  • LINEAR: Strictly linear summing of LoadingLayer results is performed

  • NONLINEAR: Zero-tension iterations are performed.

LINEAR = 1¶
NONLINEAR = 2¶
class LoadComboLateralGroupType(value)¶

Bases: Enum

The LoadingLayer type that is considered when LoadComboLayer.summing_type is LoadComboSummingType.LATERAL_GROUP.

The available values are:

  • WIND_SERVICE: Service level wind loads (corresponds to LoadingCause.WIND_SERVICE)

  • WIND_ULTIMATE: Ultimate level wind loads (corresponds to LoadingCause.WIND_ULTIMATE)

  • SEISMIC_SERVICE: Service level seismic loads (corresponds to LoadingCause.SEISMIC_SERVICE)

  • SEISMIC_ULTIMATE: Ultimate level seismic loads (corresponds to LoadingCause.SEISMIC_ULTIMATE)

SEISMIC_SERVICE = 300¶
SEISMIC_ULTIMATE = 400¶
WIND_SERVICE = 100¶
WIND_ULTIMATE = 200¶

class LoadComboSummingType(value)¶

Bases: Enum

The strategy for summing of LoadingLayer within of a LoadComboLayer.

The available values are:

  • SINGLE: The LoadComboLayer represents a single factored set of LoadingLayer

  • LATERAL_GROUP: The LoadComboLayer represents the envelope of a generated set of virtual load combinations based on the key lateral loading type. See RAM Concept manual for details.

LATERAL_GROUP = 2¶
SINGLE = 1¶
