License Usage¶

API Usage is Same as Regular Usage¶

For licensing purposes, running RAM Concept through the API is considered the same as running it through the user interface. CONNECT Edition products (such as recent releases of RAM Concept) track usage to the CONNECTED User (the person who has signed into the CONNECTION Client on the machine).

For details on the licensing system in general, see the Subscription Entitlement Service webpage.

Note - The RAM Concept Post Tension License was consolidated into the base RAM Concept License in April 2023.

Licensing Failures¶

Failure to successfully obtain a license will result in exceptions being raised by the API. The exception message will provide additional details.

The most common licensing failures are caused by:

  • No CONNECTED User signed in

  • Trial licenses expired

  • No licenses available (either all in-use, or account administrator has not allowed the CONNECTED User to use the license)

  • No network connectivity (and offline usage has been disallowed by the account administrator).

“Abandoned” Server Processes¶

Successfully creating a Concept object in the API corresponds to creating a running “server” instance of RAM Concept. This server instance consumes a license for as long as it is running. The server instance can be shut down by calling the shut_down method.

If your script crashes (or you simply forget to call shut_down), the RAM Concept server continues running, as it has not yet been told to shut down. There is an inactivity_timeout_seconds parameter in start_concept that can be used to control how long the server will run without input. The default setting for the inactivity period is one hour.

Starting another RAM Concept server (under the same CONNECTED user) while an abandoned one is still running does NOT consume an additional license. Only one license is consumed.

Troubleshooting API Licensing Issues¶

As the API uses exactly the same licensing as the user interface, the best way to troubleshoot API licensing problems is to run RAM Concept through the user interface on the same machine.
