Welcome to RAM Concept’s Scripting API!¶

RAM Concept includes a Python scripting API that allows you to automate routine RAM Concept tasks such as:

  • Creating models

  • Running analyses

  • Accessing results

Over time, we expect to expose almost all of the data and functionality in RAM Concept through the scripting API.

Python is a full-featured scripting language. By combining the features of Python with the features of RAM Concept’s API you can create some very interesting and valuable functionality. For example you could:

  • Create RAM Concept files from data in spreadsheets

  • Iterate over a variety of inputs to determine an optimal design

  • Email the optimal design to someone after it is found

  • Post results to a website

Languages other than Python¶

Currently, Python is the only scripting language that is directly supported by RAM Concept. There are many tools available to integrate Python with other languages; the best starting point for integrating Python with other languages is this web page.

Cross-Version Compatibility¶

Our intent is for future versions of RAM Concept to support past versions of the API. It is also likely that past versions of RAM Concept (8.2 and later) will support future versions of the API, but certain features in the newer API may not be supported (and may raise exceptions in the API). We expect that later APIs will almost entirely be extensions of earlier APIs, so the cost of updating scripts will be small.

2024 Feature Limitations¶

In the 2024 Release the scripting functionality is primarily limited to:

  • Creation of structures

  • Meshing

  • Adding loads and loadings

  • Adding (manual) tendons and jacks

  • Running analysis

  • Accessing reactions

  • Reading and writing files
