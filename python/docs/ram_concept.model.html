class Model(concept)¶

Bases: object

Model represents an in-memory (opened or never-saved) file in the RAM Concept process.

Model is effectively a hierarchical container of Data objects. Only the portions of the hierarchy that are available for scripting are exposed.

There can be a maximum of 1 Model associated with any Concept process at one time. There can be multiple Concept processes, so multiple Models can exist simultaneously if there is a need.

This class should never be be subclassed.

Models should ONLY be constructed via the Concept class. The methods to do that are:

  • Concept.new_model

  • Concept.open_file

Parameters:

concept (Concept) –

calc_all(timeout_seconds=None)¶

Calculate everything (except mesh, vibration and long-term deflections) in the Model.

Parameters:

timeout_seconds (int | None) – The number of seconds to wait for a response from the server (if None, a default value is used)

Return type:

None

close_model()¶

Close this Model (file) in the RAM Concept process.

The Model/file is not saved. This Model is not useful for anything after closing.

Return type:

None

generate_mesh()¶

Regenerate the mesh, using the Model’s meshing parameters. The desired element size is a property of calc_options which is accessable via Model.calc_options

Return type:

None

save_file(file_path)¶

Save this model to the given file.

Parameters:

file_path (str) – Full path to for the location to save the file.

Return type:

None

setup_new_model(code, structure)¶

Set up a fresh/blank new model for the given code and structure type.

This is intended to be called immediately after Concept.new_model returns this Model.
