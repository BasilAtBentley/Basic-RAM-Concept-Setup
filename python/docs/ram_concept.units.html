ram_concept.units module¶

class Units(uid, model)¶

Bases: Data

Units represents the singleton Units objects in a Model, which controls the user unit system.

The API operates using the same unit system as the RAM Concept UI. The methods in this class affect both the UI and the API. Generally you will want to:

  • Call get_units after opening the file, and store the return value.

  • Call either set_US_API_units or set_SI_API_units to select the unit system that you want the API to use.

  • Call set_units before closing (and saving) the file to restore the units to their original state.

For newly created files, you will want to call one of these methods to ensure the users see a reasonable set of units:

  • set_US_user_units

  • set_SI_user_units

  • set_MKS_user_units

get_units()¶

Returns the current unit settings that affect both the API and the UI.

The return value for this is only intended for use in the set_units method. The get_units/set_units pair allow you to restore the units in file after modifying them.

Return type:

str

set_MKS_user_units()¶

Set the unit system to be a reasonable default for projects using MKS units.

Return type:

None

set_SI_API_units()¶

Set the unit system to be a predictable set of SI units.

  • All lengths are meters.

  • All angles are degrees.

  • All forces are Newtons.

  • All masses are kilograms.

  • All time is seconds.

  • All temperatures are Celsius.

This leads to all areas being in square meters, all stresses being in Pascals, etc..

Return type:

None

set_SI_user_units()¶

Set the unit system to be a reasonable default for projects using SI units.

Return type:

None

set_US_API_units()¶

Set the unit system to be a predictable set of US units.

  • All lengths are inches.

  • All angles are degrees.

  • All forces are pounds.

  • All masses are pounds.

  • All time is seconds.

  • All temperatures are Fahrenheit.

This leads to all areas being in square inches, all stresses being in psi, etc..

Return type:

None

set_US_user_units()¶

Set the unit system to be a reasonable default for projects using customary US units.

Return type:

None
